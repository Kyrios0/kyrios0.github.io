强网杯 2019 线下赛 Machine Writeup

Posted on 2019-06-08 Edited on 2023-01-10
You can get the binary from here.
We found two methods to solve it.

Use overflow

This is the simpler way by using overflow in the unit test function.

The vulnerability is in systemKTest(sub_2ad8) function. If len(input) == 128, it will override part of systemKey to zero. Use this overflow with systemCheck(sub_2f6c) function, so we can override key to all zero in 5 rounds.

Finally, send 0 as key to get shell.

You can found the code of this part in exp:exp_overflow().

Use AES attack

We found the system test function is an eaiser version of 0ctf2016-people's-square

By sending the following sample and collect cipher

00 00 00 00 00 ... 00

01 00 00 00 00 ... 00

02 00 00 00 00 ... 00

03 00 00 00 00 ... 00

04 00 00 00 00 ... 00

..

FE 00 00 00 00 ... 00

FF 00 00 00 00 ... 00

We can calculate the key then get shell.

You can found the code of this part in exp:exp_aes().
